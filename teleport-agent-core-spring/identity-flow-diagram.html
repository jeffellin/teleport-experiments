üîó Identity Preserved End-to-End

The user_name claim from Teleport flows through to AgentCore, enabling audit trails and per-user authorization

Teleport: jeff ‚Üí Gateway: sub=jeff ‚Üí AgentCore: user=jeff

üîê Teleport Trust

  • User authenticates via SSO/MFA
  • Teleport issues short-lived JWT
  • JWKS at /webapi/jwks
  • Spring Gateway validates signature

üçÉ Gateway Trust

  • Mints JWTs with RS256
  • Exposes /.well-known/jwks.json
  • Sets aud for AgentCore
  • Preserves identity claims

‚òÅÔ∏è AgentCore Trust

  • Configured to trust Gateway issuer
  • Fetches JWKS to verify tokens
  • Enforces audience claim
  • Logs actions with user identity

Request Flow Summary

User ‚Üí Teleport : tsh login (SSO/MFA)
Teleport ‚Üí User : JWT {user_name: "jeff"}
User ‚Üí Gateway : GET /mcp + Authorization: Bearer <teleport-jwt>
Gateway ‚Üí Teleport : Fetch JWKS, validate JWT
Gateway ‚Üí Gateway : Mint new JWT {sub: "jeff", aud: "agentcore"}
Gateway ‚Üí AgentCore : GET /mcp + Authorization: Bearer <gateway-jwt>
AgentCore ‚Üí Gateway : Fetch JWKS, validate JWT
AgentCore ‚Üí MCP Tools : Execute as user "jeff"
